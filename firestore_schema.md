# Firestore Database Schema Design

This document outlines the Firestore collection structures and data types for the Monthly Approval Workflow application, covering `employees`, `daily_activities`, and `monthly_reports`.

---

## 1. `employees` Collection
Each document stores employee metadata, reporting hierarchy, and authentication details. We recommend using the **Employee ID (4-digit numeric string)** as the Document ID for fast lookups.

*   **Collection Path:** `/employees`
*   **Document ID:** `{employee_id}` (e.g., `"1234"`, `"2042"`)

### Document Fields:
| Field Name | Firestore Data Type | Example Value | Description |
| :--- | :--- | :--- | :--- |
| `employeeId` | `String` | `"1234"` | Unique 4-digit employee identifier. |
| `name` | `String` | `"Jane Doe"` | Full name of the employee or manager. |
| `email` | `String` | `"jane.doe@company.com"` | Main corporate email address. |
| `authEmail` | `String` | `"1234@company.com"` | Firebase Authentication email counterpart. |
| `role` | `String` | `"employee"` | Role type: `"employee"`, `"manager"`, or `"admin"`. |
| `reportingManagerId` | `String` | `"9999"` | Employee ID of their direct supervisor. |
| `reportingManagerEmail` | `String` | `"boss.manager@company.com"` | Direct supervisor's corporate email address. |
| `department` | `String` | `"Engineering"` | Core department of the employee. |
| `createdAt` | `Timestamp` | `June 23, 2026 at 12:00:00 AM UTC` | Date/time when employee record was created. |

---

## 2. `daily_activities` Collection
Stores chronologically captured workspace logs submitted by employees on a daily basis.

*   **Collection Path:** `/daily_activities`
*   **Document ID:** Auto-generated by Firestore (e.g., `da_7zJK92aX9vO12H`)

### Document Fields:
| Field Name | Firestore Data Type | Example Value | Description |
| :--- | :--- | :--- | :--- |
| `activities` | `String` | `"Implemented Compose UI tests & setup Gradle structure."` | Detailed notes about work performed on that date. |
| `date` | `Timestamp` | `June 23, 2026 at 11:55:00 PM UTC` | Date on which the described tasks were performed. |
| `employeeId` | `String` | `"1234"` | Relational key linking to the employee in the `employees` collection. |
| `monthString` | `String` | `"2026-06"` | Redundant field formatted as `YYYY-MM` for efficient querying. |
| `createdAt` | `Timestamp` | `June 23, 2026 at 5:30:12 PM UTC` | Automated timestamp when the record was written. |

#### Recommended Single/Composite Indexes:
1.  **Single Index:** `employeeId` Ascending.
2.  **Composite Index:** `employeeId` [Ascending], `date` [Descending] (Critical for aggregating current-month activities).

---

## 3. `monthly_reports` Collection
Represents monthly aggregated containers summarizing employee logs and coordinating managers/admins approval life cycle states. 

*   **Collection Path:** `/monthly_reports`
*   **Document ID:** `{employee_id}_{YYYY-MM}` (e.g., `"1234_2026-06"`) to enforce a single report per employee per month.

### Document Fields:
| Field Name | Firestore Data Type | Example Value | Description |
| :--- | :--- | :--- | :--- |
| `employeeId` | `String` | `"1234"` | Unique 4-digit employee identifier.|
| `monthYear` | `String` | `"2026-06"` | The target month represented in `YYYY-MM` string format. |
| `aggregatedActivities` | `String` | `"• June 01, 2026: Setup auth...\n• June 02, 2026: Auth testing completed..."`| Concatenated/aggregated string of all `daily_activities` notes for the month. |
| `status` | `String` | `"Pending Manager Approval"`| State machine values: `"Pending Manager Approval"`, `"Approved by Manager"`, or `"Sent to Admin"`. |
| `managerId` | `String` | `"9999"`| Key copying direct manager's employee identifier for query filtering. |
| `managerEmail` | `String` | `"boss.manager@company.com"`| Email of manager responsible for reviewing this report. |
| `submittedAt` | `Timestamp` | `June 30, 2026 at 11:55:00 PM UTC`| Automatic timestamp when the scheduled function generated this report. |
| `approvedByManagerAt` | `Timestamp` | `July 02, 2026 at 9:15:30 AM UTC` | Date/time when the manager clicked 'Approve' (if applicable). |
| `sentToAdminAt` | `Timestamp` | `July 02, 2026 at 9:15:31 AM UTC` | Date/time when Cloud Function processed status transition to Admin. |

## Security Rules Recommendations (`firestore.rules`)
```javascript
service cloud.firestore {
  match /databases/{database}/documents {
    // Only authenticated users can read employee details or submit daily logs
    match /employees/{employeeId} {
      allow read, write: if request.auth != null;
    }
    
    match /daily_activities/{activityId} {
      allow create: if request.auth != null && request.resource.data.employeeId == request.auth.token.email.split('@')[0];
      allow read: if request.auth != null && (resource.data.employeeId == request.auth.token.email.split('@')[0] || request.auth.token.email == get(/databases/$(database)/documents/employees/$(resource.data.employeeId)).data.reportingManagerEmail);
    }
    
    match /monthly_reports/{reportId} {
      allow read, write: if request.auth != null;
    }
  }
}
```
